# PayPal 开发者账号与沙盒环境注册关键截图指南

本文档提供了 PayPal 开发者账号和沙盒环境注册过程中的关键截图位置说明，帮助您更直观地完成注册流程。

## 开发者账号注册关键截图

### 1. PayPal 官网首页

**位置**: https://www.paypal.com/
**关键元素**: 
- 右上角的"注册"按钮
- 语言选择器（如果需要切换语言）

### 2. 账号类型选择页面

**位置**: 点击"注册"后的第一个页面
**关键元素**:
- "个人账户"选项
- "商家账户"选项（推荐选择此选项）

### 3. 基本信息填写页面

**位置**: 选择账号类型后的页面
**关键元素**:
- 电子邮件地址输入框
- 密码创建区域
- "继续"按钮

### 4. 个人信息填写页面

**位置**: 完成基本信息后的页面
**关键元素**:
- 姓名输入区域
- 地址信息填写区域
- 电话号码输入框
- 身份验证信息（根据国家/地区可能有所不同）

### 5. 开发者平台登录页面

**位置**: https://developer.paypal.com/
**关键元素**:
- 右上角的"Log in to Dashboard"按钮

### 6. 开发者控制台首页

**位置**: 登录成功后的页面
**关键元素**:
- 左侧导航菜单
- 欢迎信息和快速入门指南
- 右上角的个人资料图标

## 沙盒账号创建关键截图

### 1. 沙盒账号管理页面

**位置**: 开发者控制台 > 左侧菜单 "Sandbox" > "Accounts"
**关键元素**:
- "Create account"按钮
- 现有沙盒账号列表（如果有的话）

### 2. 创建沙盒账号表单

**位置**: 点击"Create account"后的弹出窗口
**关键元素**:
- Account Type 下拉菜单（Business/Personal）
- Country 下拉菜单
- Email Address 输入框（自动生成，可编辑）
- Password 输入框
- PayPal Balance 输入框
- "Create Account"按钮

### 3. 沙盒账号详情页面

**位置**: 在账号列表中点击"View/edit account"
**关键元素**:
- 账号基本信息
- "Profile"选项卡
- "Funding"选项卡（包含信用卡信息）
- "API Credentials"选项卡

## REST API 应用创建关键截图

### 1. 应用管理页面

**位置**: 开发者控制台 > 左侧菜单 "My Apps & Credentials"
**关键元素**:
- "REST API apps"部分
- "Create App"按钮
- 现有应用列表（如果有的话）

### 2. 创建应用表单

**位置**: 点击"Create App"后的页面
**关键元素**:
- App Name 输入框
- Sandbox Business Account 下拉菜单
- "Create App"按钮

### 3. 应用详情页面

**位置**: 创建应用后自动跳转的页面
**关键元素**:
- "Sandbox"部分的 Client ID 和 Secret
- "Live"部分（初始为空）
- "Sandbox App Settings"部分
- "Webhooks"部分

## Webhook 配置关键截图

### 1. Webhook 配置区域

**位置**: 应用详情页面 > 滚动到"Webhooks"部分
**关键元素**:
- "Add Webhook"按钮
- 现有 Webhook 列表（如果有的话）

### 2. 添加 Webhook 表单

**位置**: 点击"Add Webhook"后的弹出窗口
**关键元素**:
- Webhook URL 输入框
- Event types 选择区域（多选框列表）
- "Save"按钮

### 3. Webhook 详情页面

**位置**: 添加 Webhook 后的页面
**关键元素**:
- Webhook ID（用于验证）
- Webhook URL
- 已选择的事件类型列表
- "Simulate"按钮（用于测试）

## 沙盒测试关键截图

### 1. PayPal 沙盒登录页面

**位置**: 从您的应用重定向到 PayPal 支付页面
**关键元素**:

- "同意并继续"按钮
- "取消"链接

**位置**: 创建生产应用后的页面
**关键元素**:

1. **截图保存**: 建议在注册和配置过程中保存关键页面的截图，特别是包含敏感信息（如 Client ID、Secret）的页面，以便日后参考。

2. **敏感信息保护**: 在分享或存储截图时，确保遮盖敏感信息，如 Client Secret、密码等。

3. **界面变化**: PayPal 可能会不时更新其界面，实际页面可能与本文档描述略有不同。如遇差异，请以 PayPal 官方最新界面为准。

4. **多语言支持**: PayPal 支持多种语言，如果您使用的是非英文界面，按钮和菜单名称可能会有所不同，但功能位置通常保持一致。

5. **移动设备访问**: 在移动设备上访问时，界面布局可能有所不同，某些功能可能需要通过不同的导航路径访问。
